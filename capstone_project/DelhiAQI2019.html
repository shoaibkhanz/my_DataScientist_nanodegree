Delhi Air Quality 2019 Time series Analysis¶

Problem Background¶

Delhi's airquality has been extremely poor over a couple of months this year in 2019 and this is a repeated pattern which was observed in 2017 and 2018 as well. Many reasons have been attributed to the increase in air pollution and mainly it is driven by Industrial, vehicle, stuble burning. Thus, our first step should be able to better understand and create forecasts about the impact of air pollution and create awareness.

Problem Statement¶

In this analysis, we have obtained a time series dataset with following period (start: 2019-08-18, end: 2019-11-14) and I intend to predict PM2.5 feature using ARIMAX and ARIMA modelling approaches.

Data source: It was difficult to find this data. However, I have sourced it from https://openaq.org/, the recording station is IGI Terminal 3 in Delhi.

Parameter Tuining¶

The parameter tuning available in ARIMAX or ARIMA time series is available inside order argument, which takes values for p,d and q. p,d and q take it integer values as an input. The p represents the AR parameter , q represents the MA(moving average or error parameter) and lastly d is used for differencing i.e. to make non-stationary data stationary as stationarity is the fundamental underlying assumption of time series. There is also simple rule to avoid very complex model which says that p+d+q <= 5 and this condition is just guideline to have models that dont overfit the model.

(variable O3 was removed as it had a high pvalue and didnt contribute to the model)

Performance Metric¶

The metric that I have chosen for this analysis is MAPE which stands for Mean Absolute Percentage Error, while there other metrics available like MAE etc. I have used MAPE because it gives a percentage error and this helps in a simple understanding of the error. Higher MAPE suggests poort model while a smaller MAPE means a better fit of the model.

In [34]:
def mape(actual,preds):
  return np.mean(np.abs((actual - preds) / actual)) * 100

Result¶

We find that the model forecasts that in next 30 days we should expect the pollution of pm 2.5 to rise higher. Only If we had more data and preferably from the last 2 years we could have been more confident of this forecast.

We also saw that the MAPE of model1 with regressors is around ~30% which is way better than our univariate model i.e. model2, which gives us a MAPE of ~43%
